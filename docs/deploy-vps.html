Tanpa Docker

Deploy ACISPedia ke VPS Ubuntu Hostinger

Struktur repo ini sekarang memakai alur deploy native di VPS: Nginx, PHP-FPM, Composer, Node.js, queue worker, scheduler, dan Reverb sebagai proses terpisah.

1. Bootstrap VPSinstall paket dasar dan clone repo 2. Env Productionisi domain, database, Tripay, Reverb 3. Deploy Appcomposer, npm build, migrate, optimize 4. Nginx dan SSLroot ke public dan proxy websocket 5. Worker Servicesqueue, scheduler, reverb

1) Bootstrap VPS

Jalankan bootstrap sekali di VPS Ubuntu untuk menyiapkan paket utama dan clone repo.

git clone https://github.com/nnazuaff/acispedia.git
cd acispedia
sudo bash scripts/bootstrap-vps.sh

Default aplikasi ditempatkan di /var/www/acispedia kalau script dijalankan dari luar repo.

2) Siapkan Env Production

Gunakan file contoh non-Docker:

cp .env.production.example .env
nano .env

Nilai yang wajib disesuaikan:

  • APP_URL dengan domain production.
  • DB_* ke database MySQL/MariaDB di hosting atau VPS.
  • TRIPAY_* dan MEDANPEDIA_* ke kredensial production.
  • VITE_REVERB_HOST ke domain production dan REVERB_HOST=127.0.0.1.

3) Deploy / Update Aplikasi

Script deploy sekarang bekerja tanpa container.

cd /var/www/acispedia
bash scripts/deploy-on-vps.sh

Yang dilakukan script ini:

  • git pull dari branch main
  • composer install --no-dev
  • npm ci dan npm run build
  • php artisan migrate --force
  • php artisan optimize
  • queue:restart dan reverb:restart jika tersedia

4) Nginx dan SSL

Arahkan web root ke /var/www/acispedia/public. Contoh server block:

server {
    listen 80;
    server_name domainkamu.com www.domainkamu.com;
    root /var/www/acispedia/public;

    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location /app {
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_pass http://127.0.0.1:8080;
    }

    location /apps {
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_pass http://127.0.0.1:8080;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
    }
}

Setelah itu aktifkan HTTPS dengan Certbot atau panel Hostinger, lalu pastikan VITE_REVERB_SCHEME=https dan VITE_REVERB_PORT=443.

5) Queue, Scheduler, dan Reverb

Supaya proses background tetap jalan, buat service systemd terpisah.

php artisan queue:work --sleep=1 --tries=3 --timeout=90
php artisan schedule:work
php artisan reverb:start --host=127.0.0.1 --port=8080

Untuk production, jalankan ketiganya via systemd agar auto-restart saat server reboot atau proses crash.
