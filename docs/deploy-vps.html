Deploy ACISPedia ke VPS (Laravel + Vite + Reverb)

Mode: testing deploy (step-by-step)

Dokumen ini menjelaskan langkah praktis untuk upload/deploy project Laravel (dengan Vite frontend) dan realtime via Laravel Reverb ke VPS/hosting.

Checklist Cepat (untuk testing sekarang)

  • Docker (Hostinger)DNS → GitHub → docker compose Domain + SSL aktif (HTTPS) → realtime idealnya lewat WSS. Kalau pakai Cloudflare, pastikan SSL mode sesuai (Full/Full Strict).
  • App Laravel jalan via Nginx + PHP-FPM. Minimal: request ke / mengarah ke Laravel, bukan 404.

    • Docker (Hostinger) — Setup cepat

    Catatan: di Ubuntu tidak ada menu “pilih domain”. Domain dihubungkan lewat DNS. Setelah DNS mengarah ke IP VPS, barulah web server/SSL di VPS bisa memakai domain tersebut.

    1) DNS domain → arahkan ke IP VPS

    • Buat record A untuk @ (root) → IP VPS.
    • (Opsional) record A untuk www → IP VPS.
    • Tunggu propagasi DNS (biasanya menit–jam).

    2) Deploy via GitHub + docker compose

    Repo ini sudah punya Dockerfile + docker-compose.yml. Kamu bisa deploy dengan 2 cara:

    • Cara A (paling umum): SSH ke VPS → git clone → jalankan docker compose.
    • Cara B (panel Hostinger): connect GitHub → pilih repo → jalankan docker compose up. 
    # di VPS (sekali saja, jalankan sebagai root/sudo)
git clone https://github.com/nnazuaff/acispedia.git
cd acispedia
sudo bash scripts/bootstrap-vps.sh

# edit env production
cd /opt/acispedia
nano .env

# deploy / update aplikasi
bash scripts/deploy-on-vps.sh

    Jika realtime (Reverb) diproxy lewat domain HTTPS, pastikan di .env: REVERB_HOST=reverb (internal) dan VITE_REVERB_HOST=domainkamu.com, VITE_REVERB_PORT=443, VITE_REVERB_SCHEME=https (public).

    Script deploy-on-vps.sh akan generate APP_KEY langsung di file .env host jika masih kosong, lalu menjalankan build, migrate, storage link, dan optimize. Ini lebih aman untuk setup Docker karena file .env tidak ikut masuk ke image container.

    3) Auto deploy dari GitHub Actions (opsional)

    Repo ini juga punya workflow .github/workflows/deploy.yml. Setelah secrets di GitHub diisi, setiap push ke branch main bisa langsung menjalankan deploy ke VPS via SSH.

    • VPS_HOST: IP atau domain VPS.
    • VPS_PORT: biasanya 22.
    • VPS_USERNAME: user SSH di VPS.
    • VPS_SSH_KEY: private key untuk login SSH.

    Workflow akan clone repo ke ~/acispedia di VPS lalu memanggil bash scripts/deploy-on-vps.sh.

    4) SQLite di VPS: perlu diganti?

    • Bisa pakai SQLite kalau trafik kecil, single server, dan worker sedikit.
    • Disarankan ganti ke MySQL/MariaDB untuk production, apalagi kalau pakai SESSION_DRIVER=database, QUEUE_CONNECTION=database, CACHE_STORE=database (SQLite lebih rawan lock saat ada banyak proses).
    • Docker compose di atas sudah include MariaDB supaya stabil untuk queue/session/cache.
  • Asset Vite sudah dibuild → npm run build. Pastikan VITE_* sudah benar sebelum build.
  • Reverb jalan sebagai service (auto-restart). Targetnya: WebSocket lewat port 443 via reverse proxy Nginx.
  • Env production aman: APP_ENV=production, APP_DEBUG=false, secret tidak bocor. Jangan commit file .env.

Asumsi

  • OS VPS: Ubuntu 22.04/24.04 (atau sejenis).
  • Web server: Nginx.
  • PHP: 8.2+ (sesuaikan dengan requirement project).

  • Reverb berjalan lokal (mis. 127.0.0.1:8080) dan diproxy lewat Nginx port 443.

1) Install Dependensi Server

Ceklis Testing

  • Pastikan versi tool sudah benar. Contoh: php -v, composer -V, node -v, npm -v
  • Nginx aktif. Contoh: systemctl status nginx

Install tool dasar. Jalankan sebagai root / sudo.

sudo apt update
sudo apt install -y nginx git unzip curl

# PHP (contoh paket umum; sesuaikan versi PHP yang kamu pakai)
sudo apt install -y php-fpm php-cli php-mbstring php-xml php-curl php-zip php-mysql php-bcmath

# Composer
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

# Node (contoh pakai NodeSource; bisa juga pakai nvm)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs

2) Upload Code ke VPS

Direkomendasikan pakai Git (lebih gampang update). Contoh:

cd /var/www
sudo mkdir -p acispedia
sudo chown -R $USER:$USER /var/www/acispedia

git clone <URL_REPO_KAMU> /var/www/acispedia
cd /var/www/acispedia

3) Install Dependency Project

Ceklis Testing

  • Install backend dependency tanpa dev. composer install --no-dev --optimize-autoloader
  • Build frontend. npm ci lalu npm run build 
# Backend
composer install --no-dev --optimize-autoloader

# Frontend
npm ci
npm run build

4) Konfigurasi .env Production

Ceklis Testing

  • APP_URL sudah memakai HTTPS dan domain production. Contoh: APP_URL=https://domainkamu.com
  • Database bisa konek. Testing: php artisan migrate:status (harus tidak error koneksi)
  • Reverb + Vite env sesuai skema reverse proxy. Jika proxy via 443: VITE_REVERB_SCHEME=https, VITE_REVERB_PORT=443

Buat env dan isi nilai production:

cp .env.example .env
php artisan key:generate

Minimal yang perlu kamu set (contoh):

APP_NAME=ACISPedia
APP_ENV=production
APP_DEBUG=false
APP_URL=https://domainkamu.com

# Database
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=acispedia
DB_USERNAME=...
DB_PASSWORD=...

# Broadcasting / Reverb
BROADCAST_CONNECTION=reverb
REVERB_APP_ID=...
REVERB_APP_KEY=...
REVERB_APP_SECRET=...
REVERB_HOST=127.0.0.1
REVERB_PORT=8080
REVERB_SCHEME=http

# Vite (dibaca saat build)
VITE_REVERB_APP_KEY=${REVERB_APP_KEY}
VITE_REVERB_HOST=domainkamu.com
VITE_REVERB_PORT=443
VITE_REVERB_SCHEME=https

Catatan penting:

  • VITE_* harus sudah benar sebelum menjalankan npm run build.
  • Kalau kamu proxy Reverb lewat Nginx/443, maka VITE_REVERB_PORT=443 dan VITE_REVERB_SCHEME=https.
  • Kalau tidak proxy dan kamu expose port 8080 publik, maka set host/port sesuai dan pastikan firewall membolehkan.

5) Storage, Permission, Migration

Ceklis Testing

  • Permission folder writeable sudah benar. Minimal: storage/ dan bootstrap/cache bisa ditulis oleh user web server.
  • Migration sukses. php artisan migrate --force
  • Cache production dibuat. php artisan config:cache dll selesai tanpa error. 
# Permission folder yang wajib bisa ditulis
sudo chown -R www-data:www-data storage bootstrap/cache
sudo find storage bootstrap/cache -type d -exec chmod 775 {} \;

php artisan storage:link
php artisan migrate --force

# Cache untuk production
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache

6) Nginx: Serve Laravel + Proxy WebSocket Reverb

Ceklis Testing

  • Nginx config valid. sudo nginx -t
  • Website bisa diakses (HTTP/HTTPS) dan tidak 404. Contoh: curl -I https://domainkamu.com
  • WS proxy header sudah benar (Upgrade/Connection). Tanpa ini, Reverb sering gagal connect di browser.

Prinsipnya: Laravel di-handle oleh PHP-FPM, sedangkan koneksi WebSocket diarahkan ke Reverb (yang berjalan di 127.0.0.1:8080). Dengan cara ini kamu tidak perlu membuka port 8080 ke publik.

Buat file Nginx, contoh: /etc/nginx/sites-available/acispedia

server {
    listen 80;
    server_name domainkamu.com;

    root /var/www/acispedia/public;
    index index.php;

    # (Opsional) redirect ke HTTPS kalau SSL sudah aktif
    # return 301 https://$host$request_uri;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php-fpm.sock; # sesuaikan versi PHP (mis. php8.3-fpm.sock)
    }

    # Proxy WebSocket Reverb (path default biasanya /app)
    location /app {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }
}

Aktifkan site:

sudo ln -s /etc/nginx/sites-available/acispedia /etc/nginx/sites-enabled/acispedia
sudo nginx -t
sudo systemctl reload nginx

7) SSL (HTTPS) untuk WSS

Ceklis Testing

  • SSL aktif dan redirect ke HTTPS (kalau diperlukan). Target: semua traffic pakai HTTPS supaya WebSocket bisa WSS.

Direkomendasikan pakai Certbot (LetsEncrypt) atau Cloudflare. Contoh Certbot:

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d domainkamu.com

8) Jalankan Reverb sebagai Service (systemd)

Ceklis Testing

  • Service Reverb aktif. sudo systemctl status acispedia-reverb
  • Port Reverb listening (internal). Contoh: sudo ss -lntp | grep 8080

Buat file: /etc/systemd/system/acispedia-reverb.service

[Unit]
Description=ACISPedia Reverb Server
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/acispedia
ExecStart=/usr/bin/php artisan reverb:start
Restart=always
RestartSec=2

[Install]
WantedBy=multi-user.target

Aktifkan service:

sudo systemctl daemon-reload
sudo systemctl enable --now acispedia-reverb
sudo systemctl status acispedia-reverb

9) (Opsional) Queue Worker sebagai Service

Kalau kamu pakai queue, buat: /etc/systemd/system/acispedia-queue.service

[Unit]
Description=ACISPedia Queue Worker
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/acispedia
ExecStart=/usr/bin/php artisan queue:work --sleep=1 --tries=3
Restart=always
RestartSec=2

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now acispedia-queue

Troubleshooting Singkat

  • Realtime tidak jalan: cek SSL/WSS, cek Nginx proxy header Upgrade, cek service Reverb aktif.
  • Event terkirim tapi UI tidak update: pastikan VITE_REVERB_* benar dan asset terbaru sudah dibuild.
  • 403 private channel: pastikan auth route broadcasting benar dan user login/session berjalan normal.
  • Timeout WS: tambah proxy_read_timeout dan pastikan koneksi stabil.

Kalau kamu sebut detail servernya (Nginx/Apache, Ubuntu versi berapa, domain, apakah pakai Cloudflare), saya bisa sesuaikan contoh config di atas biar pas 100%.
